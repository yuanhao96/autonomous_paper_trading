# v1.3 — Expand Universe + Optimize XS Factors

## Motivation

v1.2 introduced cross-sectional factor analysis on 11 sector ETFs. Results: 0 PASS, 3 MARGINAL, 15 FAIL. The universe was too small — terciles gave only 4/3/4 stocks per group, making IC and monotonicity unreliable.

v1.3 fixes this by expanding the universe to S&P 100 (~101 stocks) and adding quintile support (~20 stocks per group). It also adds `optimize-xs` to grid-search XS factor parameters.

## Changes

### S&P 100 Universe

- `SP100_TICKERS`: Hardcoded list of ~101 large-cap S&P 100 constituents in `universe.py`
- `UNIVERSES` dict maps `"sp100"` and `"sector-etfs"` to their ticker lists
- `get_universe_tickers(name)` returns the right list by name
- Survivorship bias is a known limitation (current constituents, not point-in-time)
- Graceful download failures: individual ticker failures print a warning and skip instead of crashing

### `--universe` Flag

Added to `analyze` and `optimize-xs`:

```bash
python -m stratgen analyze --universe sp100        # Default: S&P 100
python -m stratgen analyze --universe sector-etfs  # Legacy: 11 sector ETFs
```

### Default `--n-groups` Changed to 5

With ~100 stocks, quintiles (~20 per group) are standard. Changed default from 3 to 5 for both `analyze` and `optimize-xs`.

### `optimize-xs` Command

Mirrors `factor_optimize.py` but for cross-sectional factors:

```bash
python -m stratgen optimize-xs [--reset] [--max-tries 200] [--n-groups 5]
                               [--universe {sp100,sector-etfs}]
                               [--train-end 2022-12-31] [--test-start 2023-01-01]
```

**How it works:**
1. Loads analyze results from `results_factors_xs.json`
2. Filters to PASS/MARGINAL factors with `code` and `param_ranges`
3. No-param factors are passed through with their existing metrics
4. Downloads universe, builds returns panel, splits into train/test by date
5. For each qualifying factor: grid searches params, scoring by `|mean IC|` on train
6. Best params evaluated on test set
7. Results saved to `results_factors_xs_opt.json` (with resume support)

**Scoring:** `|mean IC|` (absolute value of mean information coefficient) on the train period.

**Result structure:**
```json
{
    "factor_ref": "...",
    "name": "...",
    "optimized_params": {},
    "train_metrics": {"mean_ic": ..., "ic_t_stat": ..., "monotonicity": ..., "long_short_spread": ...},
    "test_metrics": {"mean_ic": ..., "ic_t_stat": ..., "monotonicity": ..., "long_short_spread": ...},
    "test_verdict": "PASS|MARGINAL|FAIL|ERROR"
}
```

## Usage

After upgrading to v1.3, re-run analyze on the larger universe:

```bash
# Re-analyze all 18 XS factors on S&P 100 (cached LLM code still valid)
python -m stratgen analyze --reset

# Then optimize qualifying factors
python -m stratgen optimize-xs
```

To use the old 11-ETF universe:

```bash
python -m stratgen analyze --universe sector-etfs --n-groups 3
```
